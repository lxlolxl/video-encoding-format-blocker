# Chrome Web Store Publishing Notes

## Upload package

Use the ZIP generated by:

```bash
npm run package
```

Upload `dist/video-encoding-format-blocker-1.0.0.zip`. The ZIP must contain `manifest.json` at the root.

## Suggested category

- Category: Productivity or Accessibility
- Language: English and Traditional Chinese listing text

## Privacy answers draft

Use `store/privacy-answer-draft.md` and `PRIVACY.md` as the basis for the Chrome Web Store privacy questionnaire.

## Permission justification draft

- `storage`: Stores the enabled state, selected websites, selected codec toggles, and UI language preference.
- `scripting`: Registers an early Manifest V3 content script that changes codec capability reporting on user-selected websites.
- Optional host permissions: Requested only for website match patterns entered by the user. Required so the extension can run on those selected websites.

## Review risk notes

- Keep the single purpose clear: user-selected video codec capability blocking.
- Do not add unrelated features.
- Do not add remote code or analytics.
- Avoid requesting permanent `<all_urls>` host permission unless absolutely necessary. This project uses optional host permissions instead.
- Keep `PRIVACY.md`, `DISCLAIMER.md`, and screenshots aligned with the real behavior.
